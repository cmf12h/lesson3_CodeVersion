Highlight #1 Remote Repository

Remote Repository is a repository that resides on a remote server that is shared among multiple team members. They are versions of your project that are hosted on the Internet or network somewhere.

Highlight #2 Versioning Control System

A system that automatically notes who changed the files, when they were changed, what about them was new or changed, and lastly, they merge changes to the same file.It contains the remote repository and the local repository.

Highlight #3 GIT

GIT is a distributed version control system. People like using this because every user has a complete copy of the repository data stored locally on their machine, everything is local, there is no single point of failure, processes run much faster, and it will prevent unintentional overwrite of a file on the shared repository.
